# 01. CU-BEMS 데이터셋

## 데이터셋 개요

CU-BEMS는 Chulalongkorn University의 스마트빌딩 운영 데이터셋이다.

## 데이터 특성

| 항목 | 내용 |
|---|---|
| 건물 | 7층 오피스 건물 |
| 구역 | 33개 Zone |
| 전력 데이터 | AC, 조명, 플러그 부하 |
| 환경 데이터 | 온도, 상대습도, 조도 |
| 수집 단위 | 1분 단위 |
| 기간 | 2018-07-01 ~ 2019-12-31 |
| 파일 구조 | 연도별·층별 CSV |

## 프로젝트 적용 범위

MVP에서는 2019년 데이터를 우선 사용한다.

이유:

- 1년치 데이터라 시간대/요일/계절 패턴을 분석하기 충분하다.
- 전체 18개월을 처음부터 처리하면 ETL과 DB 튜닝 부담이 커진다.
- 포트폴리오 초안에서는 2019년 데이터만으로도 충분히 기능을 보여줄 수 있다.

## 주의할 점

- 층마다 컬럼 수와 센서 구성이 다를 수 있다.
- 원본 컬럼을 그대로 서비스 테이블로 쓰면 확장성이 떨어진다.
- 전력값은 1분 단위 power 데이터이므로 kWh 집계 기준을 명확히 해야 한다.
- 결측, 고정값, 급변값이 있을 수 있으므로 품질 플래그를 둔다.

## 처리 원칙

1. 원본 행은 `raw_records.raw_payload`에 JSON으로 보존한다.
2. 표준 측정값은 `measurements`에 저장한다.
3. 대시보드와 ML은 `hourly_*` 집계 테이블을 우선 사용한다.
4. 이상탐지는 원본 측정값과 집계값을 모두 활용할 수 있다.
