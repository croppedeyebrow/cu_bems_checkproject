# 11. 완료 기준

## Phase 1 완료 기준

- FastAPI 서버 실행 가능
- `/health` API 동작
- PostgreSQL 연결 성공
- Alembic migration 실행 가능

## Phase 2 완료 기준

- ERD 기준 테이블 생성
- 주요 FK 관계 반영
- 주요 index 생성
- 샘플 insert 가능

## Phase 3 완료 기준

- CSV 파일 등록 가능
- raw_records 저장 가능
- measurements 저장 가능
- hourly aggregation 생성 가능
- ETL 재실행 시 중복 처리 방지

## Phase 4 완료 기준

- Metadata API 동작
- Dashboard API 동작
- Floor/Zone API 동작
- API 응답 타입 문서화

## Phase 5 완료 기준

- Feature table 생성
- 예측 모델 학습 가능
- prediction_results 저장 가능
- 모델 성능 조회 가능

## Phase 6 완료 기준

- 규칙 기반 이상탐지 동작
- anomaly_events 저장 가능
- 최근 경보 API 동작
- 이상 메시지 생성 가능

## Phase 7 완료 기준

- React 앱 실행 가능
- Overview 화면 표시
- Floor View 표시
- Zone View 표시
- Prediction View 표시
- Anomaly View 표시

## 최종 완료 기준

- GitHub README로 프로젝트 이해 가능
- ERD, API, ETL, ML 문서 존재
- Docker Compose로 실행 가능

- 포트폴리오 설명 문장 정리 완료
